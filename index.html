What is SNES-IDE?

SNES-IDE is an open-source, cross-platform Integrated Development Environment for creating Super Nintendo (SNES) games using pvsneslib. It simplifies the workflow for hobbyists and professionals, providing tools, templates, and automation for building, compiling, and testing SNES ROMs.

  • Languages: C (main), 65816/Spc700 assembly
  • Compilers: 816-tcc, wla-dx
  • Emulator: bsnes, snes9x
  • Cross-platform: Native on Windows, works on Linux (via Wine)
  • All-in-one: Project templates, asset converters, build scripts

Features

  • Integrated toolchain for C and assembly (816-tcc, wla-dx)
  • Project templates for quick starts
  • Batch shortcuts for editors, tools, and builds
  • Cross-platform: Native on Windows, works on Linux via Wine
  • Emulator integration (bsnes, snes9x)
  • Audio & graphics tools for SNES assets
  • Extensible and open-source

Installation Guide

Requirements

  • Operating System: Windows 10+ or Linux (with Wine 9.0+)
  • Architecture: x64, x86_64, or amd64 (not ARM)
  • Minimum Resources: 2 GB RAM, 200 MB disk space

Dependencies

  • Windows: No dependencies required.
  • Linux: Only Wine installed and configured is required.

Building from Source Dependencies

  • Windows: Python 3.8+ and .NET SDK 8.0+
  • Linux: Wine, Bash, and Linux .NET SDK 8.0+

Windows Installation

  1. Download the latest release from GitHub Releases.
  2. Decompress the .zip to a folder of your choice.
  3. Double-click INSTALL.bat to set up SNES-IDE and create shortcuts on your Desktop.

Linux Installation

  1. Download the latest release from GitHub Releases.
  2. Decompress the .zip to a folder of your choice.
  3. Install Wine: sudo apt install wine
  4. Run the configure script: 
    cd /path/to/snes-ide/
    chmod +x ./linux/configure.sh
    bash ./linux/configure.sh
  5. Follow the installer steps to generate a desktop icon or just run with ./linux/src/start.sh.

Shortcuts Generated

  • text-editor – Notepad++
  • audio-tools – Convert and manage SNES-compatible audio assets
  • graphic-tools – Convert and manage SNES-compatible graphics
  • other-tools – Access additional utilities
  • create-new-project – Scaffold a new project from a template
  • compiler – Compile your project into a SNES ROM
  • emulator – Launch your ROM in bsnes
  • snes-ide – All-in-one shortcut, the main entry of SNES-IDE

Note: Shortcuts are created in Desktop/snes-ide/ on Windows, or as a desktop entry on Linux if chosen during installation.

Getting Started

  1. Create a new project using the create-new-project shortcut or batch file.
  2. Edit your code in Notepad++ or your preferred editor.
  3. Add assets using audio-tools and graphic-tools.
  4. Compile your project with the compiler shortcut.
  5. Test your ROM in an emulator using the emulator shortcut.
  6. Or use snes-ide for an all-in-one experience.

Building from Source

  1. Clone the repo: git clone https://github.com/BrunoRNS/SNES-IDE.git
  2. Install dependencies:
    • Windows: Python 3.8+, .NET SDK 8.0+
    • Linux: Wine, Bash, .NET SDK 8.0+
  3. Follow the build instructions in the docs.
  4. After building, run the installer as described in the Installation Guide.

Verifying the Build

  • Use tree to compare the output directory with the expected structure.
  • See treeLinux.txt or treeWindows.txt for reference outputs.

Frequently Asked Questions

  • Can I use my own editor? Yes! Notepad++ is provided, but you can use any editor.
  • Does it work on Mac? Not officially, but you may try using Wine or a VM.
  • Where are my ROMs? Compiled ROMs are usually in your project folder or a Roms/ subdirectory.
  • How do I add new tools? Place them in the tools/ directory. They must be a Python or .NET tool.
  • How do I reinstall? Delete the .installed file and rerun setup.sh.
  • How do I troubleshoot? Check terminal output for errors, ensure dependencies are installed, and see the Troubleshooting section in the docs.
